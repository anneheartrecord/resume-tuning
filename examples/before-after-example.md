# 改写前后对照（示例）

下面几组取自真实简历的测试用例，已脱敏（去掉姓名、联系方式、学校等可识别信息，只保留通用的技术内容），用来演示这个 skill 实际会做什么。

## 1. 项目从「一长段流水账」到「结构化」

很多人把一个项目写成一整段，面试官扫一眼就跳过了。

**改写前**（简历里提取出来的原样）：

> 此项目基于 MIT6.824，实现了基于 Raft 协议的分布式键值存储系统，支持客户端的 Get 和 Put 请求，并且要保证数据的一致性和可靠性，此外还加入快照功能，以便快速地恢复大量数据。领导选举确保系统在任何时候都有一个正确的领导者来管理复制状态机；日志复制保证所有从节点上的日志内容相同；心跳机制通过心跳检测从节点是否存活；日志快照定期压缩日志并避免无限增长。

**改写后**：

> **分布式 KV 存储**　基于 MIT 6.824，用 Go 实现基于 Raft 的分布式键值存储，支持 Get/Put 并保证一致性与可靠性。
> - 领导选举：保证任意时刻有唯一 leader 管理复制状态机，正确处理网络分区与节点故障。
> - 日志复制：多数派确认后提交，保证各从节点日志一致。
> - 心跳 + 快照：心跳检测存活满足多数派原则；定期压缩日志做快照，避免无限增长、加速恢复。

一段话拆成「定位 + 三条要点」，标题加粗，扫读成本一下降下来。

## 2. 没有自我评价 → 生成带论据的自我评价

原简历常常根本没有自我评价，或者只有「热爱技术、积极向上」这种空话。skill 会从你已有的经历里提炼，不凭空编。

**改写前**：（无）

**改写后**：

> - **后端 + 分布式**：主攻 Go 后端，手写过基于 Raft 的分布式 KV 存储，对一致性、复制、快照有完整实现经验。
> - **计算机基础扎实**：操作系统、网络、数据结构、MySQL/Redis 机制都能讲到原理层。

## 3. 错别字 / 格式校对

PDF 里复制出来的内容经常带全角字符、反斜杠这种小毛病，自己很难发现。

**改写前**：

> 计算机网络：熟悉 TCP\IP 协议族及 HTTP 等 ⽹ 络协议；Linux：了解 Linux 常 ⽤ 命令

**改写后**：

> 计算机网络：熟悉 TCP/IP 协议族及 HTTP 等网络协议；Linux：了解 Linux 常用命令

反斜杠改成斜杠，全角字符清掉，专有名词大小写统一。

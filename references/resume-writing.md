# 简历内容写作硬标准

> 配套 `resume-standards.md`（价值观/底线）。这份给**可执行的硬指标**：每条经历写多长、强调标几处、timeline 怎么搭。
> 参考 tw93/Kami 的 resume-writing 规范，按中英文双语本地化。

---

## 一、经历用三段结构：角色 / 动作 / 结果

每段经历（项目 / 工作）拆成固定三段，下面是**字数上限**（不是目标，越短越好，意思完整就停）：

| 段 | 标签 | 写什么 | 中文上限 | 英文上限 |
|---|---|---|---|---|
| 1 | 角色 Role | 项目是什么、为什么存在、你在里面什么位置（先别上动词） | ≤60 字 | ≤40 词 |
| 2 | 动作 Action | 你用的关键决策和技术，一句一个具体做法 | ≤80 字 | ≤55 词 |
| 3 | 结果 Impact | 只放量化结果；没有数字就给规模（团队人数 / 用户量 / 流量级别） | ≤100 字 | ≤65 词 |

**自检**：把结果那句念出来。如果听着像过程描述（「优化了流水线」）而不是结果（「P95 从 800ms 降到 120ms」），重写。

> 内容少的简历不必硬凑三段，一段经历一句话也行；但凡能拆出动作→结果的，就按这个结构写。

---

## 二、强调密度：`.hl` 是手术刀不是荧光笔

`.hl`（品牌色高亮）只标**一行里最重要的那一个**数字或独特名词。

**用：**
- 每条经历最多 1 处 `.hl`，整段三行加起来最多 2 处。
- 带单位的数字：`120ms`、`38%`、`1000+ 实例`、`200K DAU`。
- 没数字时标一个独特名词：`从 0 到 1 主导`、`production-first rollout`。

**不要：**
- 标形容词 / 程度词：显著、大幅、关键、significantly。
- 同一行两个 `.hl`。
- 把一整句都高亮。

健康比例：每 80–150 字正文一处强调。满屏高亮等于没有重点。

---

## 三、timeline 是认知演进弧，不是工龄流水账（可选 section）

如果用 timeline，三步，每步一句，回答的是「你的思考/职责发生了什么转变、对读者为什么重要」，不是「哪年入职哪家」。

| 用 | 别用 |
|---|---|
| 从交付功能转向端到端对模型质量负责 | 以高级工程师身份加入 AI 团队 |
| 从写业务代码到主导内部 Agent 平台 | 2023 年入职腾讯音乐 |

内容少、或经历本身就是时间线时，**直接省掉 timeline**，别为了填充硬造。

---

## 四、按岗位调重点

| 岗位 | 重点放哪 |
|---|---|
| 技术 / 工程 | 项目 + 量化 + GitHub / 开源链接 |
| 设计 | 作品集链接 + 视觉成果 + 影响范围 |
| 产品 / 运营 / 增长 | 增长数据、留存、转化、规模 |
| 市场 / 销售 | 营收、获客成本、增长率、签单额 |
| 学生 / 应届 | 教育前置 + 项目 + 实习 + 竞赛奖项 |
| 学术 / 研究 | 教育/研究前置 + 论文 + 方法 + 引用 |

非技术岗把「技能用项目证明」换成「用成果 / 作品 / 数据证明」，原则一样：具体事实压过形容词。
